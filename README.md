# FishDrip   (FishFi)
  あなたの釣果がNFTに!?


## ■サービス概要
FishDripは、釣りを楽しみながら記録・投稿することでNFTを獲得できる、新しい“釣り×クリプト”アプリです。こちらのアプリは釣れた魚の記録を、天候・潮・場所などとともに投稿または記録し、マップ上やグラフで釣果を可視化。投稿や記録に応じてNFTやポイントを獲得できます。


## ■ このサービスへの思い・作りたい理由
私は釣りが好きで、よく魚を釣って楽しんでいました。そんな中、「どこで釣れたのか、どんな魚が釣れたのか」といった情報を記録できるアプリがあれば、もっと便利で楽しいのになと思っていました。また、大きな魚や珍しい魚が釣れたときはInstagramに投稿することもありましたが、釣りに特化した記録や共有の仕組みが少ないと感じていました。そこで、釣果を楽しく記録し、思い出として残せるアプリを作ろうと考えました。そしてもう一つ、このアプリの特徴として「釣りに応じてNFTを配布する」仕組みを取り入れたいと思ったのには理由があります。釣れて嬉しいという体験に加えて、それがNFTというデジタルな形で残ることで記念やコレクションになるといった新しい楽しみ方ができると思ったからです。さらに、NFTという仕組みはまだ日本ではあまり普及しておらず、難しそうなイメージもあります。だからこそ、釣りを通じて気軽にNFTを受け取る体験ができる場を作ることで、NFTやWeb3の世界への入り口になればいいなと考えています。NFTがもらえるから釣りに行こう！記念に残るから投稿しよう！そんなきっかけになれたら嬉しいと思います。


## ■ ユーザー層について
- **趣味で釣りを楽しんでいる人（全年齢層）**
  釣り場を投稿・共有できるので、「どこで釣れるのか気になる」「最新の釣果情報が欲しい」という方にぴったりのアプリです。釣りをしているすべての方にご利用いただけることを想定しています。

- **釣果を記録して振り返りたい人**
  「去年たくさん釣れた場所はどこだっけ？」「どのタイミングで何が釣れた？」といった記録を残しておきたいと思った経験のある方も多いのではないでしょうか。そんな振り返りをサポートするツールとして活用していただけます。

- **これから釣りを始めようか迷っている人**
  釣りに興味はあるけど「難しそう」「ハードルが高い」と感じている人にも、釣りの楽しさやアプリでの記録の価値を体験してもらいたいです。さらに「釣りをするとNFTがもらえる」という要素が、釣りを始めるきっかけになることを目指しています。

- **ブロックチェーンやNFTに興味がある人**
  Web3やNFTに興味を持っている方々にも、釣り×NFTという新しい体験を提供します。ユニークなコンセプトだからこそ、「ちょっと触ってみようかな」と感じてもらえるきっかけになると考えています。


## ■サービスの利用イメージ
ー休日ー
「今日はどこで釣りをしようかな？」と悩んだとき、FishDripを起動してみると、昨日、近くの川で釣れた投稿を発見！さっそくその場所に行ってみよう。無事に釣れたので、記録として投稿！投稿100回記念にNFTがもらえるという嬉しいサプライズも！
このように、釣りをしながら楽しく記録、さらにリワードを得ることができ、釣りでの楽しさがさらに広がります。


## ■ ユーザーの獲得について
SNSを活用し、興味を持ちそうなコミュニティに発信（自分自身のInstagram、xのアカウントを使用して投稿またはDMにて宣伝）。また釣り仲間などに共有。


## ■ サービスの差別化ポイント・推しポイント
釣果投稿・記録アプリは既にいくつか存在していますが、投稿・記録・報酬（NFT）を一体化したサービスはほとんど存在しません。また、釣り人の多くは釣果情報を共有したがらない傾向にあります。特に、釣り場の情報は貴重であり、本気で釣りを楽しんでいる人ほど非公開にするケースが多いです。FishDripでは、記録モードと投稿モードを選択可能にすることで、プライバシーと共有のバランスを保ちます。さらに、釣果の共有に対してNFTやポイントを付与することで、「情報を出すインセンティブ」を生み出す点が大きな差別化ポイントです。これにより、「いつもは秘密にしているけれど、今回は報酬がもらえるから共有してみよう」と思ってもらえるような、参加型の循環を作り出せることが、このアプリ最大の強みです。

## NFT関連についての開発の流れ
NFTの発行について、現段階で考えている流れを説明します。

- **🦊 MetaMaskウォレットアドレスの登録**
  - ユーザーがMetaMask接続 → window.ethereum.request({ method: 'eth_requestAccounts' }) でアドレス取得
    - セキュリティの観念から本人確認として署名機能（personal_sign）の導入は本リリース時予定
    -
- **NFT事前発行（Edition Drop）**
  - thirdwebのEdition Dropを使用し、画像＋レア度などのメタデータを含むNFTをあらかじめ作成
   - 配布条件や上限数などthirdwebのダッシュボードで設定
   -
- **投稿内容に応じてNFTを配布（ミント）処理**
  - 投稿数などの条件をRailsで管理し、条件達成時にマイページに「NFTを受け取る」ボタンを表示
  - フロントエンド（React + thirdweb SDK、もしくはJS）で contract.claimTo(wallet_address, tokenId, quantity) を呼び出す？ことで、ユーザー自身のウォレットにNFTをミント
  -
- **マイページでOpenSeaリンク表示**
  - ミントされたNFTはOpenSea上で確認可能なため、トークンのURLをマイページに表示してユーザーに確認してもらえるようにする予定

## NFT周りの規約などで考慮しなければならないと思う点について説明します。

- NFTの発行に伴い、ブロックチェーンネットワークの仕様上、ガス代（トランザクション手数料）が発生する場合がある点
- この際のガス代は、原則としてユーザーのご負担となる点（開発者側がガス代を払うことも可能です。その場合は必要ありませんが現時点ではどちらが負担する方針にするか決めていません。）(1回のガス代は1円程度だと予想（Polygon使用時）)
- 当方が配布するNFTは、特定の価値や利益を保証するものではない点
- ウォレットの利用・管理に関しては、ユーザーの責任のもとで行って頂く
- ウォレットの紛失や操作ミス等について、当方は一切の責任を負いとの説明。
- NFTを受け取るためには、対応するウォレット（MetaMask）をご自身でご用意いただき、接続していただく必要がある
- NFTはユーザーに無償で提供されますが、著作権は開発者に帰属
- ウォレットアドレスなどの個人情報・匿名情報をどのように取得・保存・利用するか
- NFTやウォレットの運用には、thirdwebやMetaMask、Polygon等の外部サービスを利用しています。これらのサービスの仕様変更、障害、停止等によりNFTの取得・表示・保有に影響が出る可能性がある点
